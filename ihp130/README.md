## Quickstart

```bash
# Run the estimator on your IHP130 SPICE netlist
python3 ihp130_area_estimator.py --netlist /path/to/design.spice

# With a floorplan budget
python3 ihp130_area_estimator.py --netlist /path/to/design.spice --budget "200um x 200um"

# Verbose per-device breakdown
python3 ihp130_area_estimator.py --netlist /path/to/design.spice --verbose
```

> **Note:** The database ships with PLACEHOLDER model coefficients. Results will be inaccurate until you rebuild the database using the measurement backend below.

## IHP130 SPICE Netlist Conventions

The estimator parses netlists generated by **xschem + IHP-Open-PDK**. All instances are subcircuit calls (X-prefix). Dimensions are in SI units (meters).

```spice
** MOSFETs — ng is "number of gates" (= number of fingers)
XM1 d g s b sg13_lv_nmos W=2e-6 L=0.13e-6 ng=2 m=1
XM2 d g s b sg13_hv_nmos W=4e-6 L=0.45e-6 ng=1 m=1

** Resistors (X-prefix, b=substrate bias)
XR1 n1 n2 rsil  w=0.5e-6 l=10e-6 m=1 b=0
XR2 n1 n2 rppd  w=0.5e-6 l=5e-6  m=1 b=0
XR3 n1 n2 rhigh w=1e-6   l=20e-6 m=1 b=0

** MIM Capacitors
XC1 n1 n2 cap_cmim W=10e-6 L=10e-6 m=1

** HBTs (npn13g2: fixed emitter, only nx varies)
XQ1 c b e npn13g2  Nx=2 m=1
** (npn13g2l: variable emitter length via 'le')
XQ2 c b e npn13g2l le=2e-6 Nx=2 m=1
XQ3 c b e npn13g2v le=3e-6 Nx=4 m=1

** Q-prefix is also accepted for HBTs
Q1  col base emit npn13g2 Nx=1 m=1
```

The estimator also accepts `nf` as an alias for `ng`, and both `u` and `n` SPICE unit suffixes (e.g., `W=2u`, `L=130n`).

## Mathematical Models

### MOSFET — Decoupled bounding-box model

```
Height = ah × W + bh
Width  = aw × L × nf + bw × nf + cw
Area   = Height × Width × m
```

W = width per finger (µm), L = gate length (µm), nf = finger count, m = multiplier.

### Resistor — Per-width linear model

```
Area = slope × L + intercept     (for the closest W entry in the database)
Area × m   (for parallel copies)
```

### MIM Capacitor — Border expansion model

```
Area = (W + 2×border) × (L + 2×border) × m
```

### NPN HBT `npn13g2` — Fixed emitter

```
Area = fixed_height × (aw × nx + bw) × m
```

### NPN HBT `npn13g2l` / `npn13g2v` — Variable emitter

```
Area = (ah × le + bh) × (aw × nx + bw) × m
```

All models use the "decoupled bounding-box" approach: height and width are modelled independently, then multiplied, ensuring <2.5% error even for extreme device sizes.

## Rebuilding the Database (Advanced)

### Prerequisites

- Magic VLSI **>= 8.3.573** (November 2025 release) on `$PATH`
- IHP-Open-PDK installed; `$PDK_ROOT` points to the parent of `ihp-sg13g2/`
- Python ≥ 3.10 with `numpy` and `scipy` installed

### Step 1 — Sanity check

```bash
cd tests
PDK_ROOT=/path/to/IHP-Open-PDK python3 ihp130_test_one_device.py
```

Expected output (approximate):

```
✓  PASSED  —  sg13_lv_nmos  W=0.6um  L=0.13um  nf=1
   Physical bounding box: 2.110 µm × 2.520 µm
   Area:                  5.317 µm²
```

If this fails, check your Magic version and PDK installation.

### Step 2 — Full sweep (all devices)

```bash
cd scripts
PDK_ROOT=/path/to/IHP-Open-PDK python3 ihp130_measure_devices.py
```

This performs ~350 Magic measurements (84 MOSFET points × 4 device types, plus resistors, caps, and HBTs), runs linear regression on each device, and updates `ihp130/device_db.json`.

### Step 3 — Single device (faster iteration)

```bash
python3 ihp130_measure_devices.py --device sg13_lv_nmos
python3 ihp130_measure_devices.py --device npn13g2
```

### Step 4 — Dry-run (inspect TCL without running Magic)

```bash
python3 ihp130_measure_devices.py --device cap_cmim --dry-run
```

## Key Differences vs SKY130 / GF180

| Feature | SKY130 | GF180 | **IHP130** |
|---|---|---|---|
| Device prefix | `sky130_fd_pr__*` | `gf180mcu_fd_pr__*` | `sg13_*` (short) |
| Gate finger param | `nf` | `nf` | **`ng`** in SPICE, `nf` in Magic |
| Dimension units (SPICE) | µm | µm | **SI (meters)** |
| Instance prefix | `X` | `X` | `X` (also `Q` for HBTs) |
| BiCMOS devices | BJTs (fixed macros) | — | **HBTs (parameterized)** |
| HBT emitter model | N/A | N/A | Fixed (npn13g2) or variable l (npn13g2l/v) |
| Magic RC file | `sky130A.magicrc` | `gf180mcuC.magicrc` | **`ihp-sg13g2.magicrc`** |
| Magic min version | Any | Any | **8.3.573** (Nov 2025) |

## PDK Variant Notes

The IHP130 backend targets **ihp-sg13g2** (the SG13G2 BiCMOS process). The PDK root structure expected:

```
$PDK_ROOT/ihp-sg13g2/
    libs.tech/magic/
        ihp-sg13g2.magicrc
        ihp-sg13g2.tech
        ihp-sg13g2-fet.tcl    ← MOSFET PCell definitions
        ihp-sg13g2-bjt.tcl    ← HBT PCell definitions
        ihp-sg13g2-res.tcl    ← Resistor PCell definitions
        ihp-sg13g2-cap.tcl    ← Capacitor PCell definitions
    libs.ref/
        sg13g2_pr/            ← Primitive device cells
```

The PDK is available at: [IHP-GmbH/IHP-Open-PDK](https://github.com/IHP-GmbH/IHP-Open-PDK)
